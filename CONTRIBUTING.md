# Contributing to HYBRIDJOIN Data Warehouse

Thank you for your interest in contributing! This project is an archived academic implementation demonstrating the HYBRIDJOIN algorithm for near-real-time data warehousing.

## How to Contribute

### Reporting Issues

1. Check existing issues to avoid duplicates
2. Provide a clear description with steps to reproduce
3. Include your environment details (Python version, MySQL version, OS)

### Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes with clear commit messages
4. Ensure code follows existing style conventions
5. Test your changes with the provided data files
6. Submit a PR with a clear description

### Code Style

- Follow PEP 8 for Python code
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions focused and modular

### Testing

Before submitting, verify:
- SQL scripts execute without errors on MySQL 8.0+
- Python scripts run with `python src/hybridjoin.py`
- No new warnings or errors are introduced

## Questions?

Open an issue with the `question` label for any clarifications.
